## üöÄ How to Generate a New Interview Plan

### Step 1: Provide Candidate Information

When you need a new interview plan, provide the AI with:

**Required:**
```
Candidate Name: [Name]
Interview Date: [DD-Month-YYYY]
Experience: [X.X years]
Position Level: [L1/L2/L3]
Resume/Key Projects: [Attach or describe]
```

**Optional but Recommended:**
```
L1 Interview Feedback: [Summary of L1 interview]
Specific Focus Areas: [Areas to emphasize]
Concerns to Validate: [Any red flags to probe]
```

### Step 2: AI Generates Customized Plan

The AI will:
1. Analyze the candidate's resume/profile
2. Extract key projects and achievements
3. Identify technical skills and experience level
4. Incorporate L1 feedback if provided
5. Generate customized questions based on their actual work
6. Create domain-specific coding challenges
7. Highlight areas to probe deeper

### Step 3: Review and Use

The generated plan will include:
- ‚úÖ Candidate-specific project deep-dive questions
- ‚úÖ Customized coding challenges based on their domain
- ‚úÖ Expected answers tailored to their experience level
- ‚úÖ Red flags and green flags specific to this candidate
- ‚úÖ Scoring rubric and evaluation form

## üìã Example Usage

### Example 1: Basic Request

**User:**
```
Create an interview plan for:
- Name: Sarah Johnson
- Date: November 20, 2025
- Experience: 3.5 years
- Level: L2
- Key Skills: Power BI, DAX, SQL, Python
- Main Project: Built sales analytics dashboard for 200+ users
```

**AI Will:**
- Generate plan focused on sales analytics domain
- Create SQL queries related to sales data
- Design coding challenges for sales data processing
- Adjust DAX complexity for 3.5 years experience
- Include questions about scaling to 200+ users

### Example 2: With L1 Feedback

**User:**
```
Create interview plan for:
- Name: Mike Chen
- Date: November 22, 2025
- Experience: 2 years
- Level: L2

L1 Feedback:
- Strong in Power BI visualization
```

**AI Will:**
- Add specific DAX context transition questions (probe L1 concern)
- Include detailed performance optimization questions (validate 10M records claim)
- Create finance-domain SQL queries
- Design Python challenge for report automation
- Add questions about SAP/Salesforce integration
- Note to validate advanced DAX claims

## üéØ Template Customization Points

The AI automatically customizes these sections:

### 1. Executive Summary
- Candidate background highlights (from resume)
- L1 feedback summary (if provided)
- Focus areas based on role requirements

### 2. Question Selection
- Q1: Uses candidate's actual main project
- Q2-Q4: Selected based on resume and role needs
- Coding challenges: Domain-specific scenarios

### 3. Expected Answers
- Tailored to their experience level
- References their actual projects
- Adjusted difficulty appropriately

### 4. Red Flags & Green Flags
- Specific to candidate's resume claims
- Based on L1 feedback concerns
- Role-specific critical skills

### 5. Scoring Emphasis
- Weighted based on role requirements
- Highlights must-have vs nice-to-have skills

## üìä Experience Level Guidelines

### L1 (0-2 years)
- Focus on fundamentals
- Basic DAX and SQL
- Simple data models
- Guided problem-solving

### L2 (2-4 years)
- Intermediate to advanced concepts
- Complex DAX patterns
- Performance optimization
- Independent problem-solving
- Some stakeholder management

### L3 (4+ years)
- Advanced technical depth
- Architecture decisions
- Mentoring and leadership
- Strategic thinking
- Strong stakeholder management

## üí° Tips for Best Results

1. **Provide Detailed Resume Info:** The more details about projects, the better the customization
2. **Include L1 Feedback:** Helps focus L2 interview on validating concerns
3. **Specify Role Requirements:** Mention if certain skills are critical (e.g., "Must be strong in DAX")
4. **Note Domain:** Specify industry/domain for better SQL and coding challenges
5. **Mention Scale:** Include data volumes, user counts for appropriate difficulty
